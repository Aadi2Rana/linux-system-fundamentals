## Step 1: Provisioning an Enterprise Group

In enterprise environments, access control is typically managed through groups rather than individual user permissions.

### Command Executed

```bash
sudo groupadd sysops
```

### Explanation

The `groupadd` command creates a new group named `sysops`.

This group can later be used to:

* Assign administrative responsibilities
* Manage shared resources
* Simplify permission management
* Apply role-based access control

---

## Step 2: Creating and Customizing User Accounts

A new user account was provisioned with:

* A dedicated home directory
* Membership in the `sysops` group
* Bash as the default shell

### Commands Executed

```bash
sudo useradd -m -g sysops -s /bin/bash engineer1

sudo passwd engineer1
```

### Terminal Output

```text
New password:
BAD PASSWORD: The password fails the dictionary check - it is based on a dictionary word
Retype new password:
passwd: password updated successfully
```

### Explanation

The password was intentionally configured for lab purposes despite the warning generated by the system's password quality checker.

### Flags Explained

| Flag | Purpose                    |
| ---- | -------------------------- |
| `-m` | Create a home directory    |
| `-g` | Assign primary group       |
| `-s` | Define default login shell |

Resulting home directory:

```text
/home/engineer1
```

---

## Step 3: Auditing User Account Information

System administrators routinely verify account records after user creation.

### Command Executed

```bash
getent passwd engineer1
```

### Output

```text
engineer1:x:1001:1001::/home/engineer1:/bin/bash
```

### Breakdown

| Field          | Value           |
| -------------- | --------------- |
| Username       | engineer1       |
| UID            | 1001            |
| GID            | 1001            |
| Home Directory | /home/engineer1 |
| Shell          | /bin/bash       |

### Explanation

The `getent` command retrieves account information from the system identity database.

---

## Step 4: Auditing Group Membership

### Command Executed

```bash
groups engineer1
```

### Output

```text
engineer1 : sysops
```

### Explanation

This confirms that the user `engineer1` is a member of the `sysops` group.
